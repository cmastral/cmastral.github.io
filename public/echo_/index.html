echo_

a spotify-connected rag system. connect your account — it builds a vector index of your musical identity and lets you query it conversationally.

// ingestion pipeline
// under the hood_
ingestion pipeline
→ spotify oauth 2.0 pulls your top 50 artists, top 50 tracks,
recently played, and audio features (energy, valence, tempo)
per track via the spotify web api

→ each artist is enriched by an agentic pipeline:
musicbrainz api → genres, origin, active years
wikipedia api → cultural and biographical context
claude → synthesizes a semantic description covering
mood profile, listener archetype, lyrical themes,
cultural positioning, influence network

→ enriched text chunks are embedded via openai text-embedding-3-small
and stored in an in-memory qdrant vector index scoped to your session

→ no data is persisted. your index is destroyed when you leave.
retrieval
→ queries are classified by claude into one of four types:
identity | recommendation | comparison | gap

→ each type triggers a different retrieval strategy:
identity → broad semantic search, full corpus
recommendation → filtered by mood similarity, low affinity score
comparison → targeted retrieval of named entities
gap → low-confidence retrieval surfaces blind spots

→ retrieved chunks are reranked by a second claude call
before being passed to the answer generation step
answer generation
→ claude is instructed to answer only from retrieved context,
never from prior knowledge
→ every answer cites which artists or tracks informed it
→ confidence scores are surfaced per retrieved chunk
stack
backend → fastapi on huggingface spaces (python)
vectors → qdrant in-memory (session-scoped, no persistence)
embeddings → openai text-embedding-3-small
llm → anthropic claude claude-sonnet-4-20250514
enrichment → musicbrainz api, wikipedia rest api, spotify audio features
frontend → vanilla html/js, single file, github pages
auth → spotify oauth 2.0
