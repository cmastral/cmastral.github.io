under the hood
WebGPU + WGSL shaders all 500,000 particles are updated in parallel on the GPU each frame via native WebGPU compute shaders written in WGSL — positions, velocities, life values. no WebGL, no CPU simulation, no approximations.
real-time Sobel edge detection the camera feed is processed on the CPU at 240×180 every frame using a Sobel operator. the resulting edge map is uploaded to a GPU texture and sampled by the compute shader to locate your contours.
ping-pong accumulation buffer two off-screen textures alternate each frame — one holds the previous frame faded to 97%, the other receives new particle draws on top. the result is a long-exposure photograph built in real time at 60fps.
CPU → GPU home migration the CPU samples bright pixels from the Sobel map and writes new target positions into a 4MB GPU storage buffer — 2,000 particle homes per frame, gated by stillness. the portrait builds itself over ~4 seconds of presence.
